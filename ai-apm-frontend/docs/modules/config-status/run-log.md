# 运行日志

## 页面说明

运行日志页对应 `/config/runLog`，用于查看指定宿主机已获取的 Agent 日志目录，并支持触发“获取最新日志”和下载日志文件。

## 页面结构

| 区域 | 说明 |
|------|------|
| 宿主机选择区 | `scroll-select` 选择宿主机，默认回显 `host` 查询参数 |
| 操作区 | “获取最新日志”按钮 |
| 列表区 | 日志目录、创建时间、更新时间、文件大小 |
| 下载操作 | 下载日志文件，`NEW` 状态下载后会回写已读 |

## 关键参数

| 参数 | 说明 |
|------|------|
| `host` | 当前宿主机名称 |

## 主要接口

| 接口 | 说明 |
|------|------|
| `getList` | 获取支持运行日志能力的宿主机列表 |
| `submitUpdate` | 以 `operation: 4` 触发“上传日志”任务 |
| `getAgentLogLost` | 获取当前宿主机日志列表 |
| `loadAgentLogContent` | 下载日志文件内容 |
| `updateAgentLogNew` | 下载后更新 `NEW` 状态 |

## 页面流转

1. 页面先调用 `getList` 获取可选宿主机，并根据 `host` 参数回显当前项。
2. 切换宿主机会回写路由参数并重新加载日志列表。
3. 点击“获取最新日志”后，会通过 `submitUpdate({ operation: 4 })` 触发后台任务。
4. 前端最多轮询约 4 次 Agent 状态；任务完成或超时后，重新拉取日志列表。
5. 点击下载会以 `blob` 方式获取文件，并从响应头中解析文件名。

## 注意事项

- 离线 Agent 或不支持启动/停止/重启能力的宿主机，会禁用“获取最新日志”。
- 下载接口有时会返回 JSON 错误体，页面会先尝试解析文本内容，再决定是否真正执行文件下载。
