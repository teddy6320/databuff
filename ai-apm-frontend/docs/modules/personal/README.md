# 个人中心模块

## 模块概览

个人中心对应 `/personal`，是一个非菜单静态页，主要用于查看当前登录用户信息、修改密码、维护邮箱/手机号，以及绑定和解绑钉钉、企业微信、机器人 Webhook 等通知接收方式。页面文件是 [src/views/personal/index.vue](/src/views/personal/index.vue)。

## 页面结构

| 区域 | 说明 |
|------|------|
| 基本信息区 | 用户名、角色、责任人、密码修改入口 |
| 联系方式区 | 邮箱、手机号、企业微信、钉钉、微信机器人、钉钉机器人 |
| 弹窗区 | 修改密码、绑定邮箱/手机号、编辑企业微信/钉钉配置 |

## 主要接口

| 接口 | 说明 |
|------|------|
| `getDingTalkConfig` / `getWeChatConfig` | 获取钉钉、企业微信通知通道状态 |
| `updatePwdInfo` | 修改当前用户密码 |
| `logoutHandle` | 修改密码成功后退出登录 |
| `updateAccount` | 更新邮箱或手机号 |
| `saveReceiverByType` / `unbindReceiverByType` | 绑定或解绑钉钉/企业微信接收人 |

## 页面流转

1. 页面创建时会先读取钉钉、企业微信通知通道是否已开启。
2. 点击密码编辑图标会打开 `pwd-modal`，保存成功后要求重新登录。
3. 点击邮箱或手机号的“绑定/编辑”会打开 `email-modal`，内部通过 `updateAccount` 更新当前用户资料。
4. 企业微信、钉钉的绑定逻辑依赖管理员是否已开启对应通知通道，以及当前用户是否已配置手机号。
5. Webhook 类配置复用系统管理模块中的对话框组件。

## 注意事项

- `/personal` 被路由层加入了 `__psWhiteList`，即使不是菜单页，也允许保留菜单来源参数。
- 该页强依赖当前登录态用户信息与通知配置状态，不适合脱离真实账号上下文单独调试。
- 页面中企业微信/钉钉个人绑定和机器人 Webhook 编辑是两套不同能力，文档里不要混写成同一种通知配置。
