## APM 三大特点

### ① 功能完善

覆盖应用性能监控全链路，开箱即用：

- **故障排查** — 服务红绿灯，一眼锁定异常服务
- **链路追踪** — 完整调用链，慢请求、错误一目了然
- **服务指标** — QPS、延迟、错误率、JVM 等核心指标
- **服务拓扑** — 自动绘制调用关系，快速理解系统架构

### ② 告警完善

不只是「能告警」，而是「告警好用」：

- 灵活的阈值与突变检测规则
- 事件收敛，避免告警风暴
- Webhook 通知，对接现有运维体系
- 静默计划，变更窗口不再误报

### ③ 架构极简

告别臃肿的 APM 部署：

```mermaid
flowchart LR
  Agent["应用 Agent"] --> Ingest["接入"]
  Ingest --> Doris["存储"]
  Doris --> Web["平台"]
```

**仅 3 个核心组件**（接入 + 存储 + 平台），Docker 一条命令即可跑起来。没有复杂的中间件堆砌，运维成本极低。

## AI 三大亮点

### ① 业内首个开源 AI + APM

将 LLM 能力与 APM 数据**原生融合**，不是外挂聊天框，而是真正读懂 Trace、指标、拓扑、告警的智能助手。

### ② 功能丰富

| 能力 | 能做什么 |
|------|----------|
| **智能问数** | 用自然语言查指标、Trace、拓扑、告警 |
| **服务巡检** | 自动发现异常，无需预设阈值 |
| **故障分析** | 综合多源数据，给出诊断结论 |
| **MCP 开放** | 外部 Agent 可调用平台能力 |

### ③ AI 架构先进 · 多智能体协同

```mermaid
flowchart TD
  User["你"] --> Brain["AI 大脑"]
  Brain -->|派发| Data["智能问数专家"]
  Brain -->|派发| Insp["巡检专家"]
  Data --> Result["综合回答"]
  Insp --> Result
  Result --> User
```

- **AI 大脑**统一理解意图，自动分派最合适的专家
- **数字专家**各司其职：问数、巡检、分析
- 复杂问题可**多专家并行协作**，像有一个运维团队在帮你

## 为什么选择 DataBuff

| 对比维度 | 传统 APM | DataBuff |
|----------|----------|----------|
| AI 能力 | 无或外挂 | **原生融合，读懂数据** |
| 部署复杂度 | 组件多、资源重 | **3 组件，极简部署** |
| 开源程度 | 功能阉割或商业绑定 | **AGPL 完全开源** |
| 排障方式 | 人翻图表 | **对话式智能分析** |
