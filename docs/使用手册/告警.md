# 使用手册 · 告警

## 这是什么

**主动发现问题** —— 不等用户投诉，指标异常时先记录下来。

---

## 告警能帮你做什么

| 能力 | 价值 |
|------|------|
| **阈值告警** | 错误率、延迟超线自动触发 |
| **突变检测** | 捕捉指标的突然变化 |
| **事件记录** | 保留异常发生、恢复和处理线索 |

---

## 使用流程

```mermaid
flowchart LR
  A["创建规则"] --> B["自动评估"]
  B --> C["触发告警"]
  C --> D["查看事件"]
  D --> E["AI 辅助分析"]
```

### 1. 创建规则

告警 → 规则管理 → 选择指标、设阈值、指定服务范围。

### 2. 查看事件

告警 → 事件列表，查看触发时间、服务、指标和当前状态。

### 3. 分析告警

- 在平台查看告警详情
- 跳转 AI 平台问：「这个告警什么原因？」
- 处理完毕后告警自动恢复

---

## 与 AI 协同

告警不只是「有一条记录」—— 触发后可以直接问 AI：

> 「order-service 错误率告警，帮我分析原因」

AI 会自动查指标、Trace、拓扑，给出诊断，**从告警到定位一步完成**。
